# Contributing to Clothveda

Thank you for your interest in contributing to Clothveda! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature/fix
4. Make your changes
5. Push to your fork
6. Submit a pull request

## Code Style

- Use consistent indentation (2 spaces)
- Write meaningful commit messages
- Add comments for complex logic
- Follow the existing code patterns

## Testing

Before submitting a PR:
- Ensure all tests pass
- Add tests for new features
- Check for console errors and warnings

## Pull Request Process

1. Update the README.md with any new features
2. Update documentation if needed
3. Ensure your PR description clearly describes the changes
4. Link related issues if applicable

## Reporting Issues

When reporting issues, please include:
- Description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Your environment (OS, browser, Node version, etc.)

## Code of Conduct

Please be respectful and inclusive when interacting with other contributors.
